# Money Printer 量化框架评估报告

**生成时间**: 2026-04-13
**评估范围**: 2020-01 ~ 2026-03 (75个月)
**基准**: 中证500 (ZZ500)

---

## 一、系统架构总览

```
数据层          因子层           模型层          决策层         执行层
─────────     ─────────       ─────────      ─────────     ─────────
AKShare       51个技术因子     BlendRanker     市场状态检测    反向波动率仓位
THS/Sina  →   6个基本面因子  →  (LightGBM)  →  持仓评估     →  止损/追踪止损
EM(估值)       因子表达式引擎    20日收益预测     换仓建议        组合熔断
```

**核心流程**: 每月重训模型 → 每日打分排名 → T+1开盘执行 → 风控监控

## 二、模型架构演进

### 2.1 当前生产模型: BlendRanker

| 组件 | 说明 |
|------|------|
| 主模型 (Primary) | 全样本训练，标签=excess_ret（超额收益），权重 **80%** |
| 极端模型 (Extreme) | 仅取每日涨跌幅前/后30%样本训练，权重 **20%** |
| 融合方式 | 各自预测 → 截面排名归一化 [0,1] → 加权混合 |
| 底层算法 | LightGBM 回归 (MAE) |
| 训练窗口 | 扩展窗口（expanding window），5折时序CV |
| 超参 | lr=0.05, leaves=31, min_child=50, subsample=0.8, colsample=0.8 |

**为什么选 Blend**: 主模型捕捉市场整体信号，极端模型专注尾部行为（大涨/大跌），二者互补降低过拟合。

### 2.3 关键淘汰决策逻辑

```
决策树:
1. 胜率 < 50% → 直接淘汰（不如随机）
2. 胜率 50-52% → 检查ICIR：< 0.7 淘汰，≥ 0.7 留作备选
3. 胜率 52-55% → 检查复杂度：过于复杂淘汰，简单方案留备选
4. 胜率 > 55% → 进入候选
5. 候选中选稳定性最好的（胜率波动 σ 最小）
```

**blend_best 胜出原因**:
- 胜率 55.1% (第二)，但 blend_posthoc 57.1% 的过拟合风险更高
- ICIR 0.79 稳定
- 架构简单: 仅需训练2个模型，权重固定 80/20
- 熊市仍有超额 (54.4% 胜率)

## 三、因子体系

### 3.1 51个技术因子 + 6个基本面因子

| 类别 | 数量 | 代表因子 | 作用 |
|------|------|----------|------|
| 水平因子 | 8 | RSI, MACD, 布林%B, KDJ, 动量 | 超买超卖判断 |
| 拐点因子 | 5 | RSI变化率, MACD加速度, 动量加速 | 趋势转折信号 |
| 均线偏离 | 4 | 收盘/MA5/20/60偏离, 均线排列 | 趋势强度 |
| 波动率 | 4 | ATR, 振幅, Williams%R, ADX | 波动状态 |
| 量价因子 | 7 | OBV斜率, 量比, 换手率, **Amihud非流动性** | 资金流向 |
| 收益分布 | 5 | 偏度, 峰度, 上下行波动比, 最大回撤 | 尾部风险 |
| K线形态 | 5 | 收盘位置, 上下影线, 实体比, 缺口 | 多空力量 |
| 跨域组合 | 11 | **VWAP偏离**, 布林带宽, 量价相关, CCI | 综合信号 |
| 短期动量 | 2 | 5日/10日动量 | 短线趋势 |
| **基本面** | **6** | **PE, PB, 市值对数, ROE, 营收增速, 利润增速** | **价值/成长** |

### 3.2 因子筛选: IC分析

通过截面 IC (Spearman相关系数) 和 ICIR 筛选:

| 强度 | 筛选标准 | 数量 | 代表因子 |
|------|----------|------|----------|
| 强因子 | ICIR ≥ 0.5 | 1 | **amihud_illiq** (ICIR=1.32) |
| 中等因子 | ICIR ≥ 0.3 | 1 | **vwap_dev** |
| 弱因子 | 0.15 ≤ ICIR < 0.3 | 22 | ma_alignment, rsi_14, vol_20d... |
| **淘汰因子** | **ICIR < 0.15** | **34** | **信号噪音比过低** |

**结论**: 57个因子中仅 **24个** 进入最终模型。但LightGBM对噪音因子有一定鲁棒性，全量因子在某些变体中表现也可接受。

### 3.3 被验证最强的因子

1. **Amihud非流动性** (ICIR=1.32): 低流动性股票往往有流动性溢价，是最稳定的alpha来源
2. **VWAP偏离**: 收盘价相对成交量加权均价的偏离，捕捉日内资金流向
3. **均线排列 (MA Alignment)**: MA5/MA60相对位置，经典趋势指标
4. **收盘/MA60偏离**: 中期趋势的均值回归信号

## 九、方案选择逻辑总结

### 为什么是 BlendRanker 而不是单模型?

```
单模型 (baseline):  胜率 52.7%, 超额 +3.71%
Blend (80/20):      胜率 55.1%, 超额 +3.43%  ← 胜率提升 2.4%
Blend (50/50):      胜率 45.2%, 超额 -0.33%  ← 权重错了直接亏钱
```

80/20 权重的核心逻辑: **主模型提供稳定基础(80%)，极端模型只做边际修正(20%)**。50/50 失败说明极端样本信号太窄，不能给太大权重。

### 为什么不用 LambdaRank?

```
LambdaRank: 胜率 52.9%, IC=0.069, ICIR=0.58
回归模型:   胜率 53.7%, IC=0.100, ICIR=0.76
```

LambdaRank 在排序精度上理论更优，但A股截面噪音大，回归目标更稳定。

### 为什么不用更深的树?

```
31叶 (默认): 胜率 52.7%, ICIR=0.84
64叶:        胜率 52.1%, ICIR=0.84  (无改善)
128叶:       胜率 49.5%, ICIR=0.82  (过拟合)
```

复杂度增加 → 过拟合 → 胜率下降。31叶是甜蜜点。

### 为什么不用Stage-2两阶段?

TwoStageRanker 在回测中未进入37个正式测试变体，说明早期实验中效果不佳被提前排除。可能原因: 第一阶段过滤太激进，丢失了部分alpha。

| 维度 | 评价 |
